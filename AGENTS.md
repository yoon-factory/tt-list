# Codex 작업 안내

이 저장소는 현재 **HTML 기반 Todo List** 프로젝트입니다.
백엔드가 없는 정적 프론트엔드 구조이므로, Codex는 아래 규칙을 우선합니다.

## 프로젝트 성격

- 현재 구현은 `index.html` 중심의 정적 페이지입니다.
- JavaScript와 CSS는 별도 파일로 분리되지 않았을 수 있으므로, 작업 전 실제 파일 구성을 먼저 확인합니다.
- 사용자는 백엔드 개발 관점의 추가 작업을 요청할 수 있으므로, 구조를 임의로 가정하지 않습니다.

## 기술 배경

- html
- js

## 답변 언어

- 모든 답변은 **한국어**로 작성합니다.

## 파일 수정 규칙

- 파일을 삭제하기 전에는 반드시 사용자 확인을 받습니다.
- 수정 요청 시에는 **삭제 후 새 파일 생성 방식**을 사용하지 않습니다.
- 소스코드 수정 후 한글이 깨져 보이면 인코딩을 다시 확인합니다.

## 작업 원칙

- 먼저 저장소 구조와 현재 파일을 확인한 뒤 작업합니다.
- 존재하는 파일을 우선 수정하고, 불필요한 새 파일 생성은 피합니다.
- 변경 범위는 요청한 기능에 맞게 최소화합니다.
- 정적 HTML 작업일 경우, 동작 유지와 구조 정리를 우선합니다.

## 현재 확인된 파일

- `AGENTS.md`
- `app.js`
- `index.html`

## 참고

- 이 문서는 Codex가 작업 기준을 빠르게 파악하기 위한 안내용 파일입니다.
- 프로젝트 구조가 변경되면 내용을 함께 갱신하는 것이 좋습니다.
